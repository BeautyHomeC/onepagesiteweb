Schéma de données

Ce qu'on ajoute ou modifie dans Supabase pour faire fonctionner le nouveau système.

Nouvelle table — contract_templates
-- Un template par formation × type de client
create table contract_templates (
id uuid primary key default gen_random_uuid(),
formation_id uuid references formations(id) on delete cascade,
type text not null check (type in ('particulier', 'pro')),
titre text, -- ex: "Contrat Manucure Russe Pro"
contenu text not null, -- HTML/Markdown avec {{variables}}
version int default 1,
updated_at timestamptz default now(),
unique(formation_id, type)
);
Modifications — table reservations

Nouveau

Particulier ou professionnel — détermine quel template utiliser et quels champs afficher (SIRET, raison sociale).

Valeurs étendues : en_attente_signature (contrat généré, pas encore signé), en_attente_paiement (signé, pas encore payé), confirmee, presente, annulee.
