Design · Contrat & Réservation

Flow complet — du clic « Réserver » au paiement

Voilà comment les 4 étapes s'enchaînent, et ce qui se passe côté serveur à chaque étape.

1
Infos de base + type de client
L'élève saisit : prénom, nom, email, téléphone. Elle choisit "Particulier" ou "Professionnel" (pro = champs SIRET + raison sociale s'affichent). L'ensemble est validé avant de continuer.
Client Pas encore en base
2
Contrat personnalisé — lecture obligatoire
Le contrat (particulier) ou la convention (pro) est généré à la volée avec les variables remplies : nom, formation, date, prix. Affiché dans une zone scrollable. Le bouton "Signer" n'apparaît qu'après avoir scrollé jusqu'en bas — l'élève ne peut pas sauter la lecture.
PDF généré côté serveur via API
3
Signature électronique
Deux onglets : "Taper mon nom" (champ texte) ou "Dessiner" (canvas). La signature + horodatage + IP + user-agent sont capturés. Le PDF est scellé avec ces données dans la page d'audit trail. Stocké dans Supabase Storage.
Supabase Storage
4
Paiement de l'acompte (30%) — Stripe
Redirect vers Stripe Checkout. L'ID du contrat signé est passé en metadata. Webhook Stripe → insert reservation en base (statut "confirmée") → emails envoyés : confirmation + contrat signé + programme PDF + livret d'accueil + facture d'acompte.
Stripe Checkout Insert Supabase
